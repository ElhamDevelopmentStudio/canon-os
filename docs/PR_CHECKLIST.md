# PR Checklist

Use this checklist before requesting merge.

- [ ] Scope matches SRS/SDS requirements.
- [ ] Feature is fully implemented (no placeholder logic).
- [ ] No duplicate or dead code introduced.
- [ ] Lint passes.
- [ ] Typecheck passes.
- [ ] Tests pass.
- [ ] Build passes.
- [ ] E2E passes.
- [ ] CI workflow passes on the PR branch.
- [ ] Branch is up to date with the protected base branch before merge.
- [ ] Required review approval is complete before merge.
- [ ] No direct push or force-push was used on protected branches.
- [ ] Loading/empty/error/success states are implemented.
- [ ] Accessibility basics verified (keyboard, focus, labels).
- [ ] Manual test doc added/updated in `docs/manual-tests/`.
- [ ] Commit messages use valid prefix and one-line specific format.
- [ ] No co-author or attribution trailers in commits.
